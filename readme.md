صحيح، معذرة! خطوة قاعدة البيانات (Database) هي الأهم لضمان عدم ضياع أي بيانات أثناء الانتقال.

بما أن **الجهاز الثاني** يحتوي على النسخة الأحدث من الكود، فمن المؤكد أنه يحتوي أيضاً على **أحدث نسخة من البيانات في قاعدة البيانات**.

لذلك، على الجهاز الحالي (الأول) الذي تجلس عليه الآن، سنقوم فقط بتجهيز المجلد المشترك ووضع السكربتات، لكي نستخدمها لاحقاً.

إليك الخطوات لتجهيز قاعدة البيانات والأتمتة على جهازك الحالي:

---

## 1. إنشاء مجلد المزامنة وتجاهله في Git

حتى لا نقوم برفع ملفات قاعدة البيانات الضخمة والحساسة على GitHub، سنضعها في مجلد خاص ونخبر Git بتجاهله.

1. قم بإنشاء مجلد جديد داخل مشروعك باسم `db_sync`.
2. تأكد أن هذا المجلد `db_sync` موجود داخل خدمة المزامنة السحابية الخاصة بك (مثل Dropbox أو Google Drive) ليتم نقله للجهاز الآخر تلقائياً عند فتحه.
3. افتح ملف `.gitignore` وأضف السطر التالي في نهايته (إذا لم تكن قد أضفته بعد):
```text
db_sync/

```



---

## 2. إنشاء سكربتات الأتمتة (Automation Scripts)

الآن سنقوم بإنشاء الملفين اللذين شرحناهما سابقاً داخل جذر (Root) مشروعك.

### أ. ملف `start_work.sh` (لبدء العمل مستقبلاً)

قم بإنشاء ملف بهذا الاسم وضع فيه الكود التالي:

```bash
#!/bin/bash

echo "🔄 1. جلب آخر تحديثات الكود من Git..."
git pull origin main

echo "🐳 2. تشغيل حاويات الدوكر..."
docker compose up -d

echo "⏳ الانتظار حتى تجهز قاعدة البيانات..."
sleep 5

echo "📥 3. استيراد أحدث نسخة من قاعدة البيانات..."
# استبدل db باسم الحاوية، و user و password و db_name ببياناتك الحقيقية
docker compose exec -T db mysql -u[user] -p[password] [db_name] < ./db_sync/latest_db.sql

echo "✅ جاهز للعمل! تم تحديث البيئة وقاعدة البيانات بنجاح."

```

### ب. ملف `end_work.sh` (لإنهاء العمل مستقبلاً)

قم بإنشاء ملف بهذا الاسم وضع فيه الكود التالي:

```bash
#!/bin/bash

echo "📤 1. تصدير نسخة احتياطية من قاعدة البيانات الحالية..."
# استبدل البيانات ببياناتك الخاصة
docker compose exec -T db mysqldump -u[user] -p[password] [db_name] > ./db_sync/latest_db.sql

echo "🛑 2. إيقاف حاويات الدوكر..."
docker compose down

echo "🚀 3. رفع التعديلات البرمجية إلى Git..."
git add .
read -p "أدخل رسالة التعديل (أو اضغط Enter لرسالة تلقائية): " commit_msg
if [ -z "$commit_msg" ]; then
    commit_msg="تحديث تلقائي بتاريخ $(date +'%Y-%m-%d %H:%M')"
fi

git commit -m "$commit_msg"
git push origin main

echo "🏁 تم حفظ العمل ورفعه بنجاح!"

```

---

## 3. إعطاء الصلاحيات ورفع السكربتات إلى GitHub

الآن سنعطي الملفات صلاحية التشغيل، ثم نرفعها لتكون جاهزة للجهاز الثاني:

1. في التيرمنال، أعطِ الصلاحية للأوامر:
```bash
chmod +x start_work.sh end_work.sh

```


2. ارفع السكربتات إلى GitHub:
```bash
git add start_work.sh end_work.sh .gitignore
git commit -m "Add sync automation scripts and update gitignore"
git push origin main

```



---

## 🧭 الخطة القادمة عند فتح "الجهاز الثاني":

بما أن السكربتات والتعليمات أصبحت جاهزة على GitHub، عندما تفتح **الجهاز الثاني** (صاحب النسخة الأحدث) ستفعل الآتي:

1. ستأخذ نسخة احتياطية يدوية فوراً من قاعدة البيانات الحالية هناك (لأنها الأحدث) وتضعها في مجلد `db_sync` باسم `latest_db.sql`.
2. ستقوم بعمل الـ Force Push للكود الأحدث (كما كتبنا في الـ README).

هل انتهيت من رفع السكربتات وجاهز للانتقال للجهاز الثاني لنطبق الخطوات عليه؟
